Cesium 3D Energy Dashboard and Web AR Scene

This project, developed in collaboration with a friend as part of the Master Photogrammetry and Geoinformatics module CIV at Hochschule für Technik Stuttgart, focuses on creating a 3D energy dashboard for Stöckach using Cesium and a Web AR scene with Three.js. It showcases advanced geospatial visualization and augmented reality techniques.

Cesium 3D Energy Dashboard of Stöckach

The CityGML data for this exercise was transformed into 3DTiles using FME Workbench. The CSV and CityGML data were joined using the Feature Joiner of the FME. The following figure illustrates the joining and conversion process:

The specific heating demand and the number of buildings were extracted for chart analysis in the dashboard using Chart.js and Cesium. Buildings were styled based on their energy efficiency levels in the following ranges: Below 60, 60-120, 120-170, 170-230, and Above 230 kWh/m². The charts in the dashboard were configured to be responsive to the zoom level, adjusting to include only buildings within the current zoom extent. The dashboard is shown below:

Web AR Scene

1. Initialization and Scene Setup

We started by initializing the WebXR session and setting up the Three.js scene. This includes creating a canvas element, initializing a WebGL context, and creating the scene object. The following figure illustrates the code for scene setup:

2. Cube Creation

The second key part involves creating a cube with different colored materials on each side and adding it to the scene. The materials array consisted of six different colored materials with a cube geometry of dimensions 0.5x0.5x0.5m assigned to the materials. The cube is positioned at coordinates (0, 0, -1) relative to the camera. The cube color setup is shown below:

3. Rendering Loop and Frame Handling

The final step is the rendering loop, which continuously updates and renders the scene within the AR session. “session.requestAnimationFrame(onXRFrame)” initiates the rendering loop, ensuring continuous updates. “gl.bindFramebuffer(gl.FRAMEBUFFER, session.renderState.baseLayer.framebuffer)” binds the WebGL framebuffer, and “renderer.render(scene, camera)” renders the scene.
