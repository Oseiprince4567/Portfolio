Concrete Crack Detection Web App

This is an interactive web application designed to allow users to upload images of concrete surfaces and receive immediate, AI-driven crack detection and visualization through a user-friendly interface powered by Streamlit. The app leverages a deep learning model trained on a balanced dataset to identify and highlight cracks in concrete surfaces.

Features

  • Upload Interface: Drag-and-drop upload for one or more JPG/JPEG/PNG images.
  • Preprocessing: Automatically resizes images to 120×120 pixels for model compatibility.
  • Inference: Instantly predicts the presence or absence of cracks using a trained deep learning model.
  • Visualization: Highlights cracked regions in red using OpenCV for clear visualization.
  • Batch Processing: Processes and displays results for multiple images in a single session.

Data & Methodology

The model was developed using a balanced dataset of concrete surface images processed through TensorFlow/Keras. A subset of the dataset was sampled for training to optimize computational efficiency while maintaining class balance. Key methodological steps, grounded in standard deep learning practices for image classification, included:

  • Dataset Preparation: Images were sourced from directories containing 20,000 positive (cracked) and 20,000 negative (non-cracked) examples, each 227×227 pixels in RGB format. A balanced subset of 6,000 images was randomly sampled, then split into training (70%) and testing (30%) sets using stratified sampling to preserve class distribution.
  • Data Loading and Preprocessing: TensorFlow's ImageDataGenerator was employed for pixel value rescaling to the [0, 1] range via division by 255. No data augmentation techniques (e.g., rotations, flips) were applied to evaluate baseline model performance on original data. Images were resized to 120×120 pixels during loading, with a 20% validation split from the training set. Data was batched (size 32) and flowed from Pandas dataframes for efficient iterator-based training.
  • Model Architecture: A lightweight convolutional neural network (CNN) was constructed for binary classification: Input layer (120×120×3), followed by Conv2D (16 filters, 3×3 kernel, ReLU activation), MaxPooling2D (2×2), Conv2D (32 filters, 3×3 kernel, ReLU), MaxPooling2D (2×2), GlobalAveragePooling2D to reduce spatial dimensions, and a Dense output layer (1 unit, sigmoid activation) for probability output.
  • Training Procedure: The model was compiled with Adam optimizer, binary cross-entropy loss, and accuracy metric. Training ran for up to 100 epochs on the training data, with validation monitoring. Early stopping was implemented (patience=3 on validation loss) to prevent overfitting and restore best weights. Training history was visualized to track loss convergence.
  • Evaluation: Post-training, the model was assessed on the held-out test set using loss, accuracy, confusion matrix, and classification report metrics to quantify performance in terms of precision, recall, and F1-score for each class.
  • Deployment and Inference: The trained model was saved in HDF5 format (.h5). In the Streamlit application, user-uploaded images undergo identical preprocessing (resizing to 120×120 via Pillow/NumPy), followed by model prediction. For positive detections, OpenCV applies edge detection to overlay crack regions in red. Resource efficiency is ensured by caching the model load with Streamlit's @st.cache_resource decorator.

Reflections

Building this app was a rewarding challenge. Streamlit’s rapid deployment capabilities made the UI intuitive, while TensorFlow/Keras provided a robust framework for model development. Managing large datasets and ensuring real-time inference efficiency were key hurdles, addressed through careful preprocessing and caching. Future improvements could include integrating additional crack severity metrics or enhancing visualization with interactive overlays.
